### Review Checklist

#### Universal
- [ ] PR has the `feature` label
- [ ] PR title follows `feature: <short description>` format
- [ ] Branch follows `feat/<short-description>` naming
- [ ] PR description explains what and why
- [ ] Changes follow KISS principle
- [ ] Imports are properly grouped and no unused imports exist
- [ ] Naming conventions are followed
- [ ] No commented-out or dead code
- [ ] No secrets or credentials in code
- [ ] One concern per PR (no unrelated changes bundled)
- [ ] Diff contains only changes necessary for the PR's goal (no drive-by refactors or cosmetic changes)

#### Feature-Specific
- [ ] New tests cover the added functionality
- [ ] Existing services/providers are reused where applicable
- [ ] Breaking changes are explicitly documented (if any)
- [ ] Relevant documentation updated (if public API changed)
